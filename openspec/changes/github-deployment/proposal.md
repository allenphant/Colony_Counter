## Why

將專案發佈至 GitHub 以提供更好的版本控制、開發協作基礎，並透過專業的公開展示吸引潛在使用者與貢獻者。

## What Changes

- 建立針對 Windows、Python 與 Web 開發環境優化的 `.gitignore`。
- 撰寫符合「專業正式 (Professional)」風格的專案 `README.md`。
- 使用 GitHub CLI (`gh`) 建立遠端倉庫。
- 將目前的程式碼庫推送到 GitHub。

## Capabilities

### New Capabilities
- `github-repo`: 負責遠端倉庫的初始化、認證與程式碼推送流程。
- `readme-documentation`: 建立專業的專案文件，包含安裝、使用與貢獻說明。

### Modified Capabilities
- 無

## Impact

此變更不會影響現有的專案邏輯，僅會新增 Git 相關設定檔（`.gitignore`, `.git`）與說明文件（`README.md`）。
